# Security Policy

## Reporting Security Issues

The COR Proof (Chain-of-Reasoning Proof) specification may be used in
multiple domains including legal, medical, defense, financial, and
critical infrastructure systems.

If you discover a security issue, safety concern, or misuse risk related to
this specification or its reference materials, please do not disclose it
publicly.

### Responsible Disclosure

To report a security or safety issue, please contact the maintainers privately:

- Email: security@stake-the-truth.org (placeholder)
- Subject: COR Proof Security Disclosure

Include, where possible:
- A clear description of the issue
- The affected version(s)
- Steps to reproduce (if applicable)
- Potential impact and severity
- Any suggested mitigations

We will acknowledge receipt and work toward a resolution as promptly as possible.

---

## What Qualifies as a Security Issue?

Examples include (but are not limited to):

- Validation bypasses or schema weaknesses
- Ambiguities that could enable unsafe or deceptive reasoning
- Ways to spoof evidence, citations, or confidence signals
- Failure modes in contradiction detection or escalation signaling
- Risks related to misuse in high-stakes or regulated environments
- Implementation pitfalls that could cause harm if misunderstood

---

## Non-Security Issues

Please use public GitHub Issues for:
- Specification clarifications
- Documentation improvements
- Example corrections
- Feature discussions or proposals

---

## Scope & Limitations

This repository contains only the open-source COR Proof specification and
reference materials and does not process live data, credentials, or personal information.

The following are out of scope for this security policy and should be
reported through their respective channels if applicable:

- CORIP (Chain-of-Reasoning Integrity Protocol)
- Gatehouse sanitization or claim-graph systems
- Keeper governance or escalation mechanisms
- Ledger anchoring or blockchain components
- Digital Truth Layer enterprise services
- Proprietary or commercial implementations

---

## No Bug Bounty Program

At this time, there is no formal bug bounty program associated with this repository.

We nonetheless greatly appreciate responsible disclosures that help improve
the safety and robustness of the COR Proof standard.

---

## High-Stakes Usage Disclaimer

COR Proof is a reasoning transparency and auditability standard, not a
decision-making or adjudication system.

Users deploying COR Proof in safety-critical contexts remain responsible for:
- Appropriate human-in-the-loop oversight
- Domain-specific validation
- Compliance with applicable laws and regulations
